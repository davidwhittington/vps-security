Ubuntu VPS Security Tooling

Harden your server.
In minutes, not hours.

vps-security is an open-source toolkit of bash scripts for hardening Ubuntu VPS servers. One config file, one command — firewall, SSH, fail2ban, Apache, monitoring, cert alerts, and malware defense all configured and verified.

Get started
git clone https://github.com/davidwhittington/vps-security.git
cd vps-security
cp config.env.example config.env
bash bootstrap.sh
Every script supports --dry-run — preview all changes before applying.
Ubuntu 24.04 LTS · Debian 12 compatible · Run as root

What it hardens

Run bash bootstrap.sh and get a fully hardened server covering every major security control area.

🔥

Firewall

UFW default-deny ingress, SSH/HTTP/HTTPS allowlist, connection rate limiting on ports 80 and 443.

🔑

SSH Hardening

Password auth off, modern ciphers only (AES-GCM, ChaCha20), HMAC-SHA2/ETM MACs, Curve25519 key exchange. MaxAuthTries 3.

🚫

Fail2ban

SSH + Apache jails: apache-auth, apache-badbots (1-strike, 24h ban), apache-noscript. Configurable via config.env.

🌐

Apache Hardening

ServerTokens Prod, HSTS, CSP, X-Content-Type-Options, Referrer-Policy. TRACE off, mod_status off, .git access blocked.

📊

Monitoring

Logwatch daily HTML email digest. GoAccess daily traffic report. Monthly apt upgrade + disk + fail2ban report via msmtp.

🔏

Certificate Alerts

Weekly certbot expiry monitor, independent of the monthly report. Fires when any cert is within 30 days of expiry.

🛡️

Malware Defense

ClamAV weekly web-root scan with quarantine. rkhunter rootkit scanner with clean baseline. ModSecurity + OWASP CRS.

⚙️

Kernel Hardening

sysctl: ICMP redirects off, martian logging, IP forwarding off, TCP SYN cookies, reverse path filtering enabled.

🔍

Audit Tools

9 read-only posture scripts with color-coded PASS/WARN/FAIL. Ports, SSH config, headers, AppArmor, SUID drift, and more.

↩️

Rollback

Every script backs up files before modifying. rollback.sh restores .bak files per-script or all at once.

Scripts

Twenty-one scripts split between hardening and auditing. All support --dry-run.

scripts/hardening/
01-immediate-hardening.sh UFW firewall, fail2ban (SSH + Apache jails), SSH hardening, sysctl kernel params
02-apache-hardening.sh Security headers, ServerTokens Prod, disable TRACE and mod_status, block .git access
03-setup-admin-user.sh Promote existing user to sudo, propagate SSH keys from root
04-monthly-updates-setup.sh msmtp relay configuration, monthly apt upgrade report and cron job
05-log-monitoring-setup.sh Logwatch daily digest, GoAccess traffic report, logrotate for toolkit logs
06-cert-monitor-setup.sh Weekly TLS certificate expiry alerts — fires at 30 days (configurable)
07-rkhunter-setup.sh Rootkit scanner installed with clean-system property baseline
08-auditd-setup.sh CIS Benchmark-aligned kernel audit ruleset via auditd
09-clamav-setup.sh ClamAV weekly /var/www scan with quarantine and email alert on detection
10-modsecurity-setup.sh ModSecurity + OWASP CRS in DetectionOnly mode by default
11-vhost-hardener.sh Options -Indexes, AllowOverride None, deny dotfiles and env files across all web roots
rollback.sh Restore .bak config backups from scripts 01, 02, 04, and 05 — per-script or all at once
scripts/audit/
audit.sh Full posture baseline — color-coded PASS/WARN/FAIL across all categories; --json flag available
verify.sh Post-run artifact checker — confirms every hardening script's changes took effect

ssh-audit.sh sshd configuration checker — flags weak ciphers, MACs, and KexAlgorithms
headers-check.sh Security header validator — auto-detects domains from active vhosts and checks each one

web-roots-writable.sh World-writable file and directory scanner under /var/www
suid-check.sh SUID/SGID binary baseline and filesystem drift detection

Quick start

From a fresh Ubuntu 24.04 VPS to a hardened baseline in under ten minutes.

# 1. Clone the repo on your local machine
git clone https://github.com/davidwhittington/vps-security.git
cd vps-security

# 2. Fill in your config — email, SSH port, SMTP relay
cp config.env.example config.env
nano config.env

# 3. Copy config to the server and run bootstrap
scp config.env root@YOUR_SERVER:/etc/vps-security/config.env
ssh root@YOUR_SERVER "CONFIG_FILE=/etc/vps-security/config.env bash -s" < bootstrap.sh

# 4. Verify everything took effect
bash scripts/audit/verify.sh

# 5. Run the full posture audit
bash scripts/audit/audit.sh

Not ready to commit? Pass --dry-run to bootstrap.sh to print every change that would be made without touching the system.

Private data, public tools

Audit reports, server inventories, and per-server config files contain real IPs and hostnames that shouldn't live in a public repo. vps-security uses a git submodule pattern to keep the tooling public and the server data private: 

# Public repo — hardening scripts, audit tools, docs
vps-security/
  scripts/hardening/
  scripts/audit/
  docs/
  bootstrap.sh
  config.env.example

# Private submodule — your own private repo
vps-security/private/
  servers/server1.example.com/
    config.env.local
    AUDIT_REPORT.md
  servers/inventory.yml

Clone the repo, initialize the private submodule pointing at your own private GitHub repo, and your server inventory and audit history stay yours.
