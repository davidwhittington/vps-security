capabilities

What gets hardened

01 · network
Firewall

UFW with default-deny ingress, allowlist for SSH/HTTP/HTTPS, and per-port connection rate limiting on 80 and 443.

01 · access
SSH Hardening

Password auth off. Restricts to modern ciphers (AES-GCM, ChaCha20), MACs (HMAC-SHA2/ETM), and KexAlgorithms (Curve25519, ECDH). MaxAuthTries 3, LoginGraceTime 30.

01 · intrusion
Fail2ban

SSH + Apache jails: apache-auth, apache-badbots (1-strike, 24h ban), apache-noscript. Configurable ban window and retry limits via config.env.

02 · web
Apache Hardening

ServerTokens Prod, HSTS, CSP, X-Content-Type-Options, Referrer-Policy, Permissions-Policy. TRACE off, mod_status off, .git access blocked.

05 · observability
Monitoring

Logwatch daily HTML email digest. GoAccess daily traffic report (password-protected). Monthly apt upgrade + disk + fail2ban status report via msmtp.

06 · tls
Certificate Alerts

Weekly certbot expiry monitor, independent of the monthly report. Emails on any cert expiring within 30 days (configurable via CERT_WARN_DAYS).

07–10 · malware
Malware Defense

ClamAV weekly web-root scan with quarantine. rkhunter rootkit scanner with clean-system baseline. ModSecurity + OWASP CRS (DetectionOnly by default).

01 · kernel
Kernel Hardening

sysctl: ICMP redirects disabled, martian packet logging, IP forwarding off, TCP SYN cookies, source routing disabled, reverse path filtering enabled.

audit scripts
Audit Tools

9 read-only posture scripts. Color-coded PASS/WARN/FAIL. Covers open ports, SSH config, security headers, AppArmor, SUID drift, world-writable files, and more.

rollback
Rollback

Every modifying script backs up files before touching them. rollback.sh restores .bak files per-script or all at once — with --dry-run preview.

getting started

Up and running in four commands

root@server — bash
$ git clone https://github.com/davidwhittington/vps-security.git $ cd vps-security $ cp config.env.example config.env # fill in ADMIN_EMAIL, SSH_PORT, SMTP settings $ bash bootstrap.sh

Every script supports --dry-run — pass it to bootstrap.sh to preview every change before anything is applied. Run bash scripts/audit/audit.sh afterwards to verify the full posture.

scripts

Full script inventory

scripts/hardening/
  • 01-immediate-hardening.sh UFW, fail2ban, SSH hardening, sysctl kernel params
  • 02-apache-hardening.sh Security headers, ServerTokens, TRACE off, .git block
  • 03-setup-admin-user.sh Promote user to sudo, propagate SSH keys from root
  • 04-monthly-updates-setup.sh msmtp relay, monthly apt upgrade report and cron
  • 05-log-monitoring-setup.sh Logwatch, GoAccess, logrotate for toolkit logs
  • 06-cert-monitor-setup.sh Weekly TLS certificate expiry alerts via email
  • 07-rkhunter-setup.sh Rootkit scanner with clean-system property baseline
  • 08-auditd-setup.sh CIS-aligned kernel audit ruleset via auditd
  • 09-clamav-setup.sh ClamAV weekly web-root scan with quarantine and alerts
  • 10-modsecurity-setup.sh ModSecurity + OWASP CRS, DetectionOnly by default
  • 11-vhost-hardener.sh Options -Indexes, AllowOverride None, deny dotfiles
  • rollback.sh Restore .bak config backups from scripts 01, 02, 04, 05
scripts/audit/
  • audit.sh Full posture baseline — PASS/WARN/FAIL, --json flag
  • verify.sh Post-run artifact checker — confirms every script took effect
  • ports-check.sh Unexpected listening port detector vs. allowlist
  • ssh-audit.sh sshd config and weak cipher/MAC/KexAlgorithm checker
  • headers-check.sh Security header validator across all active vhost domains
  • unattended-upgrades-check.sh Auto-update status, last-run recency, held packages
  • apparmor-check.sh AppArmor kernel module status and enforce/complain counts
  • web-roots-writable.sh World-writable file and directory scanner under /var/www
  • suid-check.sh SUID/SGID binary baseline and filesystem drift detection
